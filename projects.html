Flintstones Sega Genesis 1993 - Clone (2022)

For our first-year programming 1 course, we were tasked with applying everything we had learned to create a clone of a game of our choice. The teachers provided us with a basic framework, which we had to complete on our own to successfully recreate a 2D platformer game.

I chose to recreate The Flintstones because it presented several challenges I wanted to tackle: parallax scrolling, ledge grabbing, complex 2D animations, and more realistic physics. These aspects offered a great opportunity to deepen my understanding of C++ and the mechanics of 2D platformer games.

It was a fun project to master as much as possible of the basics of C++, and I learned a lot about 2D platformer games.

Weapon Generator Project

For this project, I developed procedural weapon generator, which provides users with the creative freedom to design their own swords and other medieval weapons.

The weapon generator allows users to construct a variety of weapons by manipulating four main components: the pommel, handle, guard, and blade. Each component can be customized using an intuitive interface.

Key Features

  • Intuitive Interface: The user-friendly interface allows for easy manipulation of weapon parts.
  • Procedural Generation: Every weapon is unique and tailored to the user's specifications.
  • Versatility: By adding extra ornaments, additional blade options, and editing the blade ramp, users can craft spears, battle axes, and more.

AI Behaviors - Flocking

C++, 10 Oct 2023

AI Behaviors - Flocking

ABOUT THE ASSIGNMENT

I was assigned to investigate flocking behavior AI in video games. The goal was to spawn as many agents as possible that have the following steering behaviors:

  • Seek / Flee
  • Arrive
  • Wander
  • Pursuit / Evade

I aimed to combine these steering behaviors to create an AI that moves agents effectively.

Because many agents with individual behaviors can cause performance loss (frame rate drop), I needed to use spatial partitioning. This technique divides space into multiple sub-spaces or partitions to aid in data processing and reduce unnecessary calculations.

In this project, I'm using flat partitioning with the following characteristics:

  • Uniform grid
  • Constant memory size
  • Cheap updates

This project will be updated to use hierarchical partitioning with a tree structure for more performance-efficient techniques.

Working of Space Partitioning

  1. Divide into a grid

  2. Check neighbors in the neighborhood radius using a check for overlapping rectangles

  3. Each neighbor checks their neighbors
Graphics - Raytracer

DirectX, C++, 12 Nov 2023

Graphics - Raytracer

ABOUT THE ASSIGNMENT

To understand Graphics Programming and how the render pipeline of graphics APIs work, we first needed to grasp the mathematics behind rasterization.

We learned how to visualize objects in screen space using the ray tracing method. This involves shooting rays from the camera view to the object to calculate the intersections with the target object.

This method is used to simulate lighting in scenes, including:

  • Reflections
  • Shadows
  • Indirect light

It is also used to determine whether we see the front or back face of an object for culling and improved performance.

Understanding these basics was essential for our next project: rasterization.

Graphics - Rasterizer

ABOUT THE ASSIGNMENT

Research - Flow Field

C++, 12 Jan 2023

Research - Flow Field

ABOUT THE ASSIGNMENT

With this project I want to introduce an exploration of flow field pathfinding as a more efficient alternative for guiding large groups of units - in this case, particles. This flow field pathfinding operates on a node grid, directing multiple units simultaneously toward a goal while dynamically avoiding obstacles. This method not only streamlines the movement of large groups but also ensures the shortest possible path is taken, significantly enhancing efficiency in situations where coordinated movement is very important.

3D - Weapon Generator

Houdini, 30 May 2024

3D - Weapon Generator

About the Assignment

For this project, we were tasked with developing a procedural 3D model generator. I decided to focus on creating a weapon generator, which provides users with the creative freedom to design their own swords and other medieval weapons.

Project Overview

The weapon generator allows users to construct a variety of weapons by manipulating four main components: the pommel, handle, guard, and blade. Each component can be customized using an intuitive interface, where parameters can be adjusted to create unique and intricate designs.

Key Features

  • Intuitive Interface: Easy manipulation of weapon parts through sliders and input fields.
  • Procedural Generation: Produce countless variations, ensuring unique and tailored designs.
  • Versatility: Design various weapon types including swords, spears, and battle axes.
  • Real-time Preview: Immediate visual feedback for precise adjustments.

Customization Options

  • Pommel: Adjust the shape, size, and decoration of the pommel.
  • Handle: Customize the length, grip texture, and materials of the handle.
  • Guard: Modify the guard's width, curvature, and design elements.
  • Blade: Choose from various blade shapes, and adjust the blade's length and width.
